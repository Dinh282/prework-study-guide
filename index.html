HTML

  • HTML stands for Hypertext Markup Language.
  • Has a head element that stores information that is not shown to user.
  • The body and footer contains information that is visible to the user.
    • Texts
    • URL Links
    • Media Files (images, videos, audios)

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS code is responsible for the presentation of a Webpage (colors, layout, fonts, spacing, size, etc.
  • There are three ways add a style sheet: inline, internal, and external.
  • HTML must be linked to a CSS file with the link element when using an external CSS.
  • CSS ruleset includes the selector and declaration(includes the property and property value).

Git

  • A free and open source version control system.
  • Tracks changes to the source code.
  • Allows numerous developers to work on the same project.
  • Git Commands
    • git status: checks what branch we are currently on.
    • git checkout -b branch-name: creates a new branch and switches to it.
    • git commit -m "message": commits all staged work.
    • git commit --amend: allows modifications of most recent commit.
    • git clone (transfer protocol): copies a repository to a new location.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function holds a set of code to be executed once called upon.
  • Functions need to be declared before they can be called.
  • An array can store many data of the same or different types.
  • Methods are like built-in JS functions. They are actions done on objects.
